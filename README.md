


# Sign Language Translation & Display System (v1)

An IoT and web-based application designed to process sign language data and interface with an OLED hardware display. This repository contains the application server, hardware testing scripts, and automation utilities.

---

## 📂 Project Structure

| File / Folder | Description |
| :--- | :--- |
| **`app.py`** | The main application script (likely a Flask/FastAPI backend or Streamlit interface) handling core sign language processing or API routes. |
| **`oled_test.py`** | Hardware testing script to verify connections, text rendering, or graphical outputs on an attached OLED display module. |
| **`requirements-oled.txt`** | Python dependencies specifically required for hardware interfacing, display drivers, and application libraries. |
| **`start.sh`** | Shell script to automate the startup sequence of the application or environment initialization. |
| **`cron_log.log`** | Log file generated by a cron job, used to monitor background automation, uptime, or errors. |

## 🚀 Getting Started

### 1. Prerequisites
Ensure you have Python 3.x installed on your system (or Raspberry Pi / microcontroller if interfacing with physical hardware).

### 2. Installation
Clone the repository and install the required dependencies:
```bash
git clone [https://github.com/sarveshmule1504/sign_langv1.git](https://github.com/sarveshmule1504/sign_langv1.git)
cd sign_langv1
pip install -r requirements-oled.txt

```

### 3. Running the Hardware Test

To verify that your OLED display is wired correctly and functioning, run:

```bash
python oled_test.py

```

### 4. Starting the Main Application

You can launch the main server using the provided shell script or by executing `app.py` directly:

```bash
# Via shell script
chmod +x start.sh
./start.sh

# Or directly via Python
python app.py

```

## ⏰ Automation & Logging

The project includes a `cron_log.log` file, indicating that it can be scheduled as a background service via `cron`.

To set up the application to run automatically on boot or intervals:

1. Open your crontab configuration: `crontab -e`
2. Add a line pointing to your `start.sh` script, routing the output to the log file:
```bash
@reboot /path/to/sign_langv1/start.sh >> /path/to/sign_langv1/cron_log.log 2>&1

```
